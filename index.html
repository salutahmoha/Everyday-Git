Everyday Git

What is Git

Git is a widely-used version control system (VCS) that helps developers manage and track changes to their code. A VCS allows multiple people to collaborate on a project without conflicts and keeps a history of all changes, making it easy to revert to previous versions if needed. While there are other VCS options like Subversion (SVN) and Mercurial, this blog will focus on Git and its popular companion, GitHub, to explore how they enhance software development and collaboration.

Installing Git

To check if Git is installed on your machine, run the following command:

git --version

If not installed:

  • Windows: Download from Git for Windows.
  • macOS: Install via Homebrew with brew install git.
  • Linux: Install via package manager (e.g., sudo apt-get install git).

Configuring git

To configure your username and email,you need to run the commands below

Configure your username with: git config --global user.name "Your Name"

Configure your email with: git config --global user.email "your.email@example.com"

Adding files to staging area

The staging area is where changes are prepared before committing. Tracked files are being monitored by Git, while untracked files are not yet added to the repository.

Add files with: git add filename

Commiting changes

Commit changes with git commit -m "Your commit message".

Write clear and concise commit messages that describe the changes made.

View past commits using git log and revert to a specific commit if needed.

Branches

Branches are independent lines of development. They allow you to work on different features simultaneously.

Creating a branch

Create a new branch with git branch branch_name.

Viewing All Branches

View all branches with git branch -a.

The active branch is marked with an asterisk (*).

Switching active branches

Switch branches with git checkout branch_name or git switch branch_name.

Merging a branch

Merge changes from one branch to another using git merge branch_name.

Deleting a branch

Delete a branch with git branch -d branch_name.

if the branch you want to delete contains some info: delete using git branch -D branch_name

Recommend learning about GitHub for remote repositories or exploring advanced Git features like rebasing and stashing.
